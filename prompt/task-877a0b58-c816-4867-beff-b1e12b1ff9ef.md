# [منطق] عدم ثبت و ردیابی execution context در ai_manager

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `ai_llm` یک ناسازگاری منطقی پیدا شد:

prompt_helper دارای قابلیت ثبت اجرای پرامپت‌ها (PromptExecution) است، اما ai_manager هیچ logging یا tracking مشابهی برای انتخاب مدل، fallbackها و نتایج validation ندارد.

## 💥 پیامد (impact)
اشکال‌زدایی pipeline بسیار دشوار می‌شود و نمی‌توان فهمید کدام مدل در کدام مرحله چه خروجی‌ای تولید کرده است.

## 🛠 پیشنهاد رفع اولیه
یک execution context object به ai_manager اضافه کنید که شامل: مدل انتخاب شده، تعداد retryها، نتایج validation، و timestamp باشد. این context باید در دیتابیس ذخیره شود.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
