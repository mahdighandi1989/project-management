# [منطق] عدم وجود permission/authorization در pipeline

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

در مستندات ارائه‌شده، هیچ‌یک از کامپوننت‌ها (database, model, service) شامل مکانیزم‌های permission یا authorization نیستند. مسیرهای mutation (مانند آپلود فایل در oversight_upload_session) بدون هیچ بررسی دسترسی قابل اجرا هستند.

## 💥 پیامد (impact)
هر کاربری می‌تواند بدون احراز هویت یا مجوز، فایل آپلود کند، سشن ایجاد کند یا به داده‌های حساس دسترسی یابد. این یک vulnerability امنیتی جدی است.

## 🛠 پیشنهاد رفع اولیه
یک middleware یا dependency برای احراز هویت و authorization به pipeline اضافه کنید. مثلاً در endpointهای FastAPI از Depends(get_current_user) استفاده کنید و نقش‌های دسترسی (مثلاً admin, inspector) را بررسی کنید.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
