### نحوه خواندن این متغیرها در کد بک‌اند:

این متغیرها پس از تعریف در پنل هاستینگ، در محیط اجرایی (runtime environment) برنامه شما در دسترس قرار می‌گیرند. برای خواندن آن‌ها در کد، بسته به زبان برنامه‌نویسی، از روش‌های استاندارد زیر استفاده می‌شود:

#### در پایتون (Python):

در پایتون، با استفاده از ماژول `os` می‌توانید به این متغیرها دسترسی پیدا کنید.

```python

# خواندن کلید API مربوط به OpenAI
openai_api_key = os.environ.get('OPENAI_API_KEY')

# خواندن توکن ربات تلگرام
telegram_token = os.environ.get('TELEGRAM_BOT_TOKEN')

# خواندن شناسه چت تلگرام
telegram_chat_id = os.environ.get('TELEGRAM_CHAT_ID')

```

```
import os
from typing import List, Dict

class Settings(BaseSettings):
    # ...
    TELEGRAM_BOTS: List[Dict[str, str]] = []

    def __init__(self, **values):
        super().__init__(**values)
        self._load_telegram_bots()

    def _load_telegram_bots(self):
        i = 0
        while True:
            token = os.environ.get(f'TELEGRAM_BOT_TOKEN_{i}')
            chat_id = os.environ.get(f'TELEGRAM_CHAT_ID_{i}')
            if token and chat_id:
                self.TELEGRAM_BOTS.append({'token': token, 'chat_id': chat_id})
                i += 1
            else:
                break
```

**Scope:** این مرحله، زیرساخت یک راه‌حل حرفه‌ای و دینامیک را فراهم می‌کند که به ایده کاربر مبنی بر مدیریت از طریق «فرانت‌اند» نزدیک است. اقدام اصلی، طراحی یک اسکیمای پایگاه داده (مثلاً یک جدول `bots` با ستون‌های `id`, `name`, `token`, `chat_id`, `is_active`) و راه‌اندازی آن است. سپس کد بک‌اند باید اصلاح شود تا به جای خواندن از متغیرهای محیطی، لیست ربات‌های فعال را از این جدول بخواند. این کار پیکربندی ربات‌ها را به طور کامل از فرآیند دیپلوی جدا می‌کند.
**Excerpt:**
```
ایا میشه در فرانت این کارو کرد و ایدی و توکن تلگرام های جدید رو اونجا ثبو ذخیره کرد تا ربات اجازه کار در اون تلگرام ها رو پیدا کنه؟

راهکار «مدیریت دینامیک» این مشکل را با استفاده از روش‌های مقیاس‌پذیرتر (مانند ذخیره توکن‌ها در پایگاه داده) حل می‌کند.
```

### Step 4: ایجاد یک API امن در بک‌اند برای مدیریت ربات‌ها (CRUD)
**Status:** `pending` (0%)
**Scope:** بر اساس پایگاه داده مرحله قبل، در این مرحله باید Endpointهای یک API امن برای مدیریت (Create, Read, Update, Delete) پیکربندی ربات‌ها در بک‌اند ایجاد شود. برای مثال: `POST /api/admin/bots` برای افزودن ربات جدید، `GET /api/admin/bots` برای لیست کردن ربات‌ها و `DELETE /api/admin/bots/:id` برای حذف یک ربات. امنیت در این مرحله حیاتی است و این Endpointها باید حتماً با استفاده از مکانیزم‌های احراز هویت و مجوز دسترسی (Authentication/Authorization) محافظت شوند تا فقط ادمین به آن‌ها دسترسی داشته باشد.
**Excerpt:**
```
ایا میشه در فرانت این کارو کرد و ایدی و توکن تلگرام های جدید رو اونجا ثبو ذخیره کرد تا ربات اجازه کار در اون تلگرام ها رو پیدا کنه؟
```

### Step 5: ساخت پنل ادمین (فرانت‌اند) برای مدیریت توکن‌ها
**Status:** `pending` (0%)
**Scope:** این مرحله، چشم‌انداز کاربر برای مدیریت از طریق یک رابط کاربری را محقق می‌کند. اقدام اصلی، ساخت یک صفحه وب ساده (فرانت‌اند) به عنوان پنل ادمین است که با API امن ساخته شده در مرحله قبل تعامل دارد. این پنل به ادمین اجازه می‌دهد تا پس از ورود، لیست ربات‌های موجود را ببیند، ربات جدیدی را با وارد کردن توکن و شناسه چت اضافه کند و ربات‌های موجود را حذف نماید. این روش، مدیریت ربات‌ها را بدون نیاز به دستکاری کد، متغیرهای محیطی یا دیپلوی مجدد، بسیار آسان و امن می‌کند.
**Excerpt:**
```
ایا میشه در فرانت این کارو کرد و ایدی و توکن تلگرام های جدید رو اونجا ثبو ذخیره کرد تا ربات اجازه کار در اون تلگرام ها رو پیدا کنه؟

ملاحظات امنیتی و محل ذخیره‌سازی توکن‌ها (بک‌اند در مقابل فرانت‌اند)
```
