# [منطق] عدم validation برای extra_data در inspector_session

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

فیلد extra_data به صورت JSON ذخیره می‌شود، اما هیچ validation schema برای آن تعریف نشده است. هر داده دلخواهی می‌تواند ذخیره شود که ممکن است باعث ناسازگاری یا حملات تزریق شود.

## 💥 پیامد (impact)
ذخیره داده‌های مخرب یا ناسازگار در دیتابیس که می‌تواند باعث crash برنامه یا سوءاستفاده امنیتی شود.

## 🛠 پیشنهاد رفع اولیه
یک Pydantic model برای validation ساختار extra_data تعریف کنید و قبل از ذخیره، داده را اعتبارسنجی کنید. از JSON Schema یا Marshmallow نیز می‌توان استفاده کرد.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
