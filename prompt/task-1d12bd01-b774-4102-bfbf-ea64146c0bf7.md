# [منطق] نبود مکانیزم fallback در prompt_helper.py برای کش نامعتبر

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `ai_llm` یک ناسازگاری منطقی پیدا شد:

prompt_helper.py از کش استفاده می‌کند (use_cache=True) اما در صورت invalid شدن کش یا تغییر دیتابیس، مکانیزمی برای invalidate کردن خودکار کش یا fallback به دیتابیس پیش‌بینی نشده است.

## 💥 پیامد (impact)
احتمال استفاده از پرامپت‌های قدیمی یا نامعتبر تا زمان expire دستی کش

## 🛠 پیشنهاد رفع اولیه
اضافه کردن timestamp به کش و بررسی consistency با دیتابیس در هر درخواست، یا پیاده‌سازی cache invalidation بر اساس رویدادهای تغییر در دیتابیس.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
