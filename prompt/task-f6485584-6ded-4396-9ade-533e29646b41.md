# افزودن قابلیت pin و ویرایش عنوان در تسک‌ها

## Raw Idea

ارتقای صفحهٔ مرکز نظارت (تب تسک‌ها، آرشیو، گزارش‌ها) از یک لیست تخت به یک workspace قابل‌مدیریت با:
- صفحه‌بندی قابل‌تنظیم با state در backend
- سورت غنی و فیلتر چند-بعدی context-aware
- جستجوی live بدون لگ روی همهٔ فیلدهای متنی
- پین تسک‌ها در active + archive جداگانه
- مدیریت کامل عنوان: تولید بهتر + ویرایش دستی + بازنگری در هر verify
- **layout بهتر کارت تسک** (دو ردیف دکمه + جلوگیری از overflow)
- persistence کامل در backend

### قواعد
- پیش‌فرض: **۱۰۰ آیتم در هر صفحه**
- گزینه‌ها: `25 | 50 | 100 | 200 | 500` (هر تب page_size مستقل ذخیره می‌کند)
- nav: `« اول | < قبلی | صفحه N از M | بعدی > | آخر »` + "برو به صفحه N"
- نمایش: `نمایش X-Y از Z تسک` (روی نتیجهٔ فیلتر‌شده، نه کل)
- pagination همیشه روی **نتیجهٔ بعد از فیلتر/سورت/جستجو** اعمال می‌شود
- بعد از pin/unpin/delete/edit، صفحهٔ فعلی حفظ می‌شود
- اگر صفحهٔ فعلی خالی شد → reset به صفحهٔ ۱

## 📝 ۶. مدیریت عنوان

### ۶.۱ تولید بهتر در idea→prompt
- explicit instruction در system prompt:
  > **عنوان**: حداکثر ۸ کلمه با **فعل عملیاتی + موضوع مشخص + scope روشن**. مثال خوب: "افزودن دکمهٔ undo به super-task". مثال بد: "بهبود سیستم"، "تغییرات"
- regex validator: کلمات generic (`بهبود/تغییر/سیستم/update/fix/improve`) بدون context → **retry** با hint قوی‌تر
- حداکثر ۲ retry، سپس fallback

### ۶.۲ Regenerate — flow کامل
1. Backend: پرامپت جدید + عنوان جدید
2. response:
```json
{"new_prompt": "...", "new_title": "...", "old_title": "...", "title_changed": true}
```
3. اگر title_changed و !manual_override → dialog:
   > 🔄 عنوان پیشنهادی جدید:  
   > **قبلی**: "X"  
   > **پیشنهاد جدید**: "Y"  
   > [نگه داشتن قدیمی] [قبول جدید] [ویرایش دستی]
4. اگر manual_override → فقط prompt به‌روز

### ۶.۳ ویرایش دستی
- ✏️ کنار عنوان → inline input (Enter ذخیره، Esc لغو)
- `PATCH /tasks/{id}` با `{"title": "..."}`
- ست `manual_title_override = True`
- `title_history` به‌روز
- دکمهٔ "🔓 بازکردن قفل عنوان" برای reset manual_override

### تاریخچهٔ عنوان (UI)
- accordion `📜 تاریخچهٔ عنوان (N)`
- entries: `ts [source]` — `old_title` → `new_title`
- منبع: `manual | ai_generate | verify_reassess | regenerate | consolidation`

### ۶.۴ Re-evaluation در **هر verify** ⭐

**قاعده:** هر بار verify (single/bulk، fast/deep) → یک‌بار title reassess

#### الگوریتم
1. پایان `verify_task`
2. اگر `manual_title_override == True` → skip
3. AI call سبک (`DEFAULT_EXTRACTION_MODEL_ID`، ~۱۰۰ توکن، ~۴s):
   - ورودی: عنوان فعلی + AC + completed/remaining + verification_status + last_summary
   - پرسش: "آیا عنوان دقیق است؟ اگر نه، پیشنهاد جدید. اگر بله، `keep`."
   - خروجی: `{action: "keep"|"update", new_title?, reason?}`
4. اگر `update` و new_title متفاوت:
   - عنوان به‌روز + `title_history` entry با `source: "verify_reassess"`
   - badge کوچک "✨ به‌روز شد"

#### Cost
- ۲۰۰ تسک × ~۴s ÷ ۳ موازی = ~۴.۵ دقیقه اضافه در bulk verify
- این هزینه صریحاً پذیرفته شده (نه conditional)

#### در consolidation
- بعد از build super-task، یک پاس reassess جداگانه (تا عنوان generic AI cluster اصلاح شود)

## 🎨 ۱۳. بهبود Layout و Responsive کارت تسک ⭐ **جدید**

### مشکلات فعلی (مشاهده شده در screenshot ها):
1. در صفحات تنگ (لپ‌تاپ کوچک، split window، responsive)، **عنوان به‌صورت عمودی wrap می‌شود** و فضای زیادی می‌گیرد (۸-۱۰ خط برای ۳-۴ کلمه)
2. **متن چک‌لیست از مرز کارت بیرون می‌زند** (overflow horizontal)
3. **دکمه‌های action** (AI با اجرا، verify، من اعمال کردم، تاریخچه، بازتولید، کپی پرامپت، …) همه در یک ردیف افقی، فضای زیادی از عرض کارت را اشغال می‌کنند → فضای کمتری برای محتوا

### راه‌حل

#### الف) دکمه‌های action در دو ردیف زیر هم

```
┌────────────────────────────────────────────┐
│ ⏸ 📁 👁 ⚠ │ [AI با اجرا] [verify] [من اعمال کردم]│  ← ردیف ۱
│ [تاریخچه] [بازتولید] [کپی پرامپت]            │  ← ردیف ۲
└────────────────────────────────────────────┘
```

- ردیف ۱: action های اصلی/پربسامد (اجرا، verify، apply)
- ردیف ۲: action های ثانویه (تاریخچه، regenerate، کپی)
- جدا کردن status icons (⏸ 📁 👁 ⚠) به سمت چپ
- استفاده از `flex-wrap` تا اگر صفحه باز بود در یک ردیف، اگر تنگ بود در دو ردیف
- gap یکنواخت بین دکمه‌ها (`gap-2`)
- اندازهٔ دکمه‌ها یکسان (`min-w-[80px]` یا `flex-1` در حالت تنگ)

#### ب) عنوان: max-width و truncate هوشمند

- max-width مطلق: `max-w-full break-words`
- **اگر عنوان > ۸۰ کاراکتر**: truncate با ellipsis + tooltip متن کامل + کلیک = expand
- line-height بهینه (`leading-relaxed`)
- استفاده از `text-overflow: ellipsis` در حالت کم‌فضا

#### ج) چک‌لیست: overflow کنترل شده

- `overflow-wrap: break-word` + `word-break: break-word` روی متن آیتم‌ها
- هر آیتم چک‌لیست: `max-w-full` با `pl-2` برای checkbox
- اگر متن > ۲ خط: collapse با "بیشتر..." (`line-clamp-2`)
- کلیک روی "بیشتر..." → expand inline
- اعداد و فلش‌ها (مرحله ۱، باقی‌مانده، …) sticky position نشن، فقط text wrap درست

#### د) Responsive breakpoints

```css
/* mobile + narrow */
@media (max-width: 768px) {
  .task-card-buttons { flex-direction: column; gap: 0.5rem; }
  .task-card-title { font-size: 0.875rem; }
}
/* tablet */
@media (max-width: 1024px) {
  .task-card-buttons-row { flex-wrap: wrap; }
}
/* desktop wide */
@media (min-width: 1280px) {
  .task-card-buttons-row { flex-wrap: nowrap; }  /* یک ردیف اگر جا داشت */
}
```

#### ه) Card container

- `min-width: 280px` (جلوگیری از خیلی تنگ)
- `padding: 0.75rem 1rem` به‌جای padding متفاوت
- `box-sizing: border-box` (پیش‌فرض React، فقط مطمئن شویم)
- `overflow-x: hidden` روی container کارت (تا child ها bleed نکنند)

#### و) Test responsive

DevTools → Device toolbar → تست با:
- `1920×1080` (desktop)
- `1366×768` (laptop)
- `1024×768` (small laptop)
- `768×1024` (tablet portrait)
- `375×667` (mobile)

در همه باید:
- ✅ عنوان قابل خواندن، wrap درست
- ✅ دکمه‌ها در دو ردیف یا چندردیف بدون bleed
- ✅ چک‌لیست در داخل کارت، بدون overflow
- ✅ scroll عمودی روی card مجاز، scroll افقی غیرمجاز

### ۶.۴ Re-evaluation در **هر verify** ⭐

**قاعده:** هر بار verify (single/bulk، fast/deep) → یک‌بار title reassess

#### الگوریتم
1. پایان `verify_task`
2. اگر `manual_title_override == True` → skip
3. AI call سبک (`DEFAULT_EXTRACTION_MODEL_ID`، ~۱۰۰ توکن، ~۴s):

## 🎨 ۱۳. بهبود Layout و Responsive کارت تسک ⭐ **جدید**

### مشکلات فعلی (مشاهده شده در screenshot ها):
1. در صفحات تنگ (لپ‌تاپ کوچک، split window، responsive)، **عنوان به‌صورت عمودی wrap می‌شود** و فضای زیادی می‌گیرد (۸-۱۰ خط برای ۳-۴ کلمه)
2. **متن چک‌لیست از مرز کارت بیرون می‌زند** (overflow horizontal)
3. **دکمه‌های action** (AI با اجرا، verify، من اعمال کردم، تاریخچه، بازتولید، کپی پرامپت، …) همه در یک ردیف افقی، فضای زیادی از عرض کارت را اشغال می‌کنند → فضای کمتری برای محتوا

### راه‌حل

#### الف) دکمه‌های action در دو ردیف زیر هم

```
┌────────────────────────────────────────────┐
│ ⏸ 📁 👁 ⚠ │ [AI با اجرا] [verify] [من اعمال کردم]│  ← ردیف ۱
│ [تاریخچه] [بازتولید] [کپی پرامپت]            │  ← ردیف ۲
└────────────────────────────────────────────┘
```

- ردیف ۱: action های اصلی/پربسامد (اجرا، verify، apply)
- ردیف ۲: action های ثانویه (تاریخچه، regenerate، کپی)
- جدا کردن status icons (⏸ 📁 👁 ⚠) به سمت چپ
- استفاده از `flex-wrap` تا اگر صفحه باز بود در یک ردیف، اگر تنگ بود در دو ردیف
- gap یکنواخت بین دکمه‌ها (`gap-2`)
- اندازهٔ دکمه‌ها یکسان (`min-w-[80px]` یا `flex-1` در حالت تنگ)

#### ب) عنوان: max-width و truncate هوشمند

- max-width مطلق: `max-w-full break-words`
- **اگر عنوان > ۸۰ کاراکتر**: truncate با ellipsis + tooltip متن کامل + کلیک = expand
- line-height بهینه (`leading-relaxed`)
- استفاده از `text-overflow: ellipsis` در حالت کم‌فضا

#### ج) چک‌لیست: overflow کنترل شده

- `overflow-wrap: break-word` + `word-break: break-word` روی متن آیتم‌ها
- هر آیتم چک‌لیست: `max-w-full` با `pl-2` برای checkbox
- اگر متن > ۲ خط: collapse با "بیشتر..." (`line-clamp-2`)
- کلیک روی "بیشتر..." → expand inline
- اعداد و فلش‌ها (مرحله ۱، باقی‌مانده، …) sticky position نشن، فقط text wrap درست

#### د) Responsive breakpoints

```css
/* mobile + narrow */
@media (max-width: 768px) {
  .task-card-buttons { flex-direction: column; gap: 0.5rem; }
  .task-card-title { font-size: 0.875rem; }

@media (max-width: 1024px) {

}
/* desktop wide */
@media (min-width: 1280px) {
  .task-card-buttons-row { flex-wrap: nowrap; }  /* یک ردیف اگر جا داشت */
}
```

#### ه) Card container

- `min-width: 280px` (جلوگیری از خیلی تنگ)
- `padding: 0.75rem 1rem` به‌جای padding متفاوت
- `box-sizing: border-box` (پیش‌فرض React، فقط مطمئن شویم)
- `overflow-x: hidden` روی container کارت (تا child ها bleed نکنند)

#### و) Test responsive

- `1920×1080` (desktop)
