# Duplicate health check endpoints in model_profiles.py and models.py

## Raw Idea

Both `backend/app/api/routes/model_profiles.py` (line 130-139) and `backend/app/api/routes/models.py` (line 78-79, 162-163) define `/health` and `/providers` endpoints that serve similar purposes. The `model_profiles.py` health endpoint returns `db_available`, `models_available`, `profiler_available` flags, while `models.py` has `/providers` returning provider availability. This duplication causes confusion and maintenance overhead. Additionally, `model_profiles.py` has a prefix `/api/models` (line 50) while `models.py` has prefix `/models` (line 15), creating two different URL paths for similar functionality.
