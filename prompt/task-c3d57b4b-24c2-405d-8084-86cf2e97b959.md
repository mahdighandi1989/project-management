- `backend/app/services/notification_service.py` — این سرویس مسئول ارسال اعلان‌ها، از جمله اعلان‌های یادآوری (بر اساس کامیت `67441ae`) است. پس از پیاده‌سازی API مدیریت یادآوری، این سرویس باید بتواند از `ReminderService` برای بازیابی یادآوری‌های فعال استفاده کند.
- `backend/app/models/project.py` — یادآوری‌ها باید به پروژه‌ها مرتبط شوند. این فایل مدل `Project` را تعریف می‌کند و نیاز به اضافه کردن یک `relationship` به مدل جدید `Reminder` دارد.
- `backend/app/core/database.py` — تمام عملیات دیتابیس در `ReminderService` و Endpoints API از `SessionLocal` و `get_db` که در این فایل تعریف شده‌اند، استفاده خواهند کرد.
- `backend/app/services/background_scheduler.py` — اگر یادآوری‌ها نیاز به زمان‌بندی برای ارسال اعلان داشته باشند، این سرویس (که از APScheduler استفاده می‌کند) مسئول اجرای دوره‌ای وظایف برای بررسی یادآوری‌های سررسید شده و فراخوانی `notification_service` خواهد بود.

## 🌐 نقشهٔ وابستگی‌ها
مدل `Reminder` (در `backend/app/models/reminder.py`) به عنوان موجودیت اصلی برای یادآوری‌ها عمل می‌کند و یک کلید خارجی به `Project` (در `backend/app/models/project.py`) خواهد داشت. `ReminderService` (در `backend/app/services/reminder_service.py`) منطق کسب‌وکار برای مدیریت این مدل را فراهم می‌کند و به `backend/app/core/database.py` برای تعامل با دیتابیس وابسته است. روتر `reminders` (در `backend/app/api/routes/reminders.py`) این سرویس را فراخوانی می‌کند و خود توسط `backend/app/main.py` در برنامه اصلی FastAPI گنجانده می‌شود. `backend/app/services/notification_service.py` که مسئول ارسال اعلان‌های تلگرام برای یادآوری‌ها است، به `ReminderService` وابسته خواهد بود تا یادآوری‌های فعال را بازیابی کند.

## 🔍 Context و وضعیت فعلی
کاربر درخواست ایجاد یا توسعه Endpoints جدید در API بک‌اند را برای مدیریت «یادآوری‌ها» (reminders) دارد. هدف اصلی پیاده‌سازی عملیات کامل CRUD (ایجاد، خواندن، به‌روزرسانی، حذف) بر روی این یادآوری‌ها است. این شامل Endpoints برای ثبت یادآوری جدید، ویرایش یادآوری‌های موجود، مشاهده لیست یادآوری‌ها و ارتباط آن‌ها با پروژه‌ها است. کاربر به صراحت اشاره کرده که این مرحله شامل پیاده‌سازی منطق ربات تلگرام نیست، اما در بخش مربوط از درخواست اصلی خود، به نیاز تلگرام اشاره کرده: «در تلگرام وقتی میخوام ذیل یه پروژه یاداوری ثبت کنم چیزی در دکمه ها و زیر دکمه ها نمیبینم... نحوه ارتباط یادآوری با پروژه‌ها در تلگرام (سناریوی فعلی و مورد انتظار)». این نشان می‌دهد که این Endpoints برای پشتیبانی از قابلیت‌های مدیریت یادآوری در رابط کاربری تلگرام ضروری هستند. کلیدواژه‌های اصلی درخواست شامل `API Endpoints`, `CRUD`, `یادآوری`, `بک‌اند`, و `پروژه‌ها` هستند.

با بررسی تاریخچه کامیت‌ها، کامیت `67441ae feat(reminder): add reminder task type with Telegram firing + checklist` نشان می‌دهد که مفهوم «reminder task type» و مکانیزم «Telegram firing» برای یادآوری‌ها از قبل در پروژه وجود دارد. این به این معنی است که بخش ارسال و زمان‌بندی اعلان‌های یادآوری احتمالاً در `backend/app/services/notification_service.py` و `backend/app/api/routes/notifications.py` پیاده‌سازی شده است. وظیفه فعلی، ایجاد لایه API برای *مدیریت* (ایجاد، ویرایش، حذف) این یادآوری‌ها است که `notification_service` می‌تواند از آن برای دسترسی به داده‌های یادآوری استفاده کند. ساختار فعلی پروژه با دایرکتوری‌های `backend/app/api/routes/` برای Endpoints، `backend/app/models/` برای مدل‌های دیتابیس و `backend/app/services/` برای منطق کسب‌وکار، برای این توسعه مناسب است. مدل `Project` نیز در `backend/app/models/project.py` (که در ساختار پروژه ذکر شده) باید با مدل جدید `Reminder` ارتباط داشته باشد.

## 🪜 مراحل اجرایی پیشنهادی
1. برای پیاده‌سازی Endpoints API مدیریت یادآوری‌ها، مراحل زیر پیشنهاد می‌شود:

1.  **ایجاد مدل و شمای Pydantic برای یادآوری:**
    *   یک فایل جدید به نام `backend/app/models/reminder.py` ایجاد شود.
    *   در این فایل، یک مدل SQLAlchemy به نام `Reminder` تعریف شود که شامل فیلدهایی مانند `id` (کلید اصلی)، `project_id` (کلید خارجی به `projects.id` برای ارتباط با پروژه)، `message` (متن یادآوری)، `due_date` (تاریخ و زمان سررسید)، `status` (وضعیت یادآوری، مثلاً 'pending', 'completed', 'cancelled')، `created_at` و `updated_at` باشد.
    *   شمای Pydantic مربوطه (`ReminderCreate`, `ReminderUpdate`, `ReminderSchema`) برای اعتبارسنجی درخواست‌ها و پاسخ‌های API تعریف شود.

2.  **ایجاد سرویس `ReminderService`:**
    *   یک فایل جدید به نام `backend/app/services/reminder_service.py` ایجاد شود.
    *   کلاس `ReminderService` پیاده‌سازی شود که منطق کسب‌وکار برای عملیات CRUD روی مدل `Reminder` را در بر گیرد. این سرویس متدهایی مانند `create_reminder`, `get_reminder`, `get_reminders_by_project`, `update_reminder`, و `delete_reminder` را ارائه خواهد داد و با سشن دیتابیس (`db: Session`) کار می‌کند.

3.  **پیاده‌سازی روتر API برای یادآوری‌ها:**
    *   یک فایل جدید به نام `backend/app/api/routes/reminders.py` ایجاد شود.
    *   یک `APIRouter` برای مسیر `/reminders` تعریف شود.
    *   Endpoints زیر پیاده‌سازی شوند:
        *   `POST /reminders/`: برای ایجاد یک یادآوری جدید، با ورودی `ReminderCreate` و خروجی `ReminderSchema`.
        *   `GET /reminders/{reminder_id}`: برای دریافت جزئیات یک یادآوری خاص، با خروجی `ReminderSchema`.
        *   `GET /reminders/project/{project_id}`: برای دریافت لیست یادآوری‌های مرتبط با یک پروژه خاص، با قابلیت فیلتر بر اساس `status`، با خروجی `List[ReminderSchema]`.
        *   `PUT /reminders/{reminder_id}`: برای به‌روزرسانی یک یادآوری موجود، با ورودی `ReminderUpdate` و خروجی `ReminderSchema`.
        *   `DELETE /reminders/{reminder_id}`: برای حذف یک یادآوری.

4.  **ادغام روتر جدید در برنامه اصلی:**
    *   فایل `backend/app/main.py` ویرایش شود تا روتر `reminders` جدید را import کرده و آن را به برنامه اصلی FastAPI اضافه کند.

5.  **برقراری ارتباط با مدل `Project`:**
    *   فایل `backend/app/models/project.py` (در صورت وجود) ویرایش شود تا یک `relationship` با مدل `Reminder` اضافه شود، که امکان دسترسی آسان به یادآوری‌های یک پروژه را فراهم کند و یکپارچگی ارجاعی را تضمین کند.

6.  **بازبینی `NotificationService`:**
    *   فایل `backend/app/services/notification_service.py` بازبینی شود تا اطمینان حاصل شود که می‌تواند از `ReminderService` جدید برای بازیابی و پردازش یادآوری‌های زمان‌بندی شده برای ارسال اعلان‌های تلگرام استفاده کند، و با قابلیت «reminder task type» موجود همسو باشد.

  ```python
  logger = logging.getLogger(__name__)
  
  REMINDER_TEXT, REMINDER_TIME, REMINDER_PROJECT = range(3)
  
  class TelegramBotService:
      def __init__(self):
          self.application = Application.builder().token(settings.TELEGRAM_BOT_TOKEN).build()
          self._register_handlers()
          self.user_states = {}
  
      def _register_handlers(self):
          self.application.add_handler(CommandHandler("start", self.start))
          conv_handler = ConversationHandler(
              entry_points=[CommandHandler("reminder", self.start_reminder_conversation)],
              states={
                  REMINDER_TEXT: [MessageHandler(filters.TEXT & ~filters.COMMAND, self.get_reminder_text)],
                  REMINDER_TIME: [MessageHandler(filters.TEXT & ~filters.COMMAND, self.get_reminder_time)],
                  REMINDER_PROJECT: [CallbackQueryHandler(self.select_project_for_reminder)]
              },
              fallbacks=[CommandHandler("cancel", self.cancel_conversation)]
          )
          self.application.add_handler(conv_handler)
  
      async def start_reminder_conversation(self, update: Update, context):
          self.user_states[update.effective_user.id] = {}
          await update.message.reply_text("لطفاً متن یادآوری را وارد کنید:")
          return REMINDER_TEXT
  
      async def handle_update(self, update_data: dict):
          """Process incoming Telegram update from webhook."""
          update = Update.de_json(update_data, self.application.bot)
          await self.application.process_update(update)
  ```

کاربر درخواست دارد که قابلیت ربط دادن یک یادآوری به یک پروژه خاص در ربات تلگرام پیاده‌سازی شود. در حال حاضر، وقتی کاربر می‌خواهد «ذیل یه پروژه یاداوری ثبت کنم»، «چیزی در دکمه ها و زیر دکمه ها نمیبینم» که امکان انتخاب پروژه را فراهم کند. این مشکل نشان می‌دهد که منطق ارتباط یادآوری با پروژه‌ها در تلگرام (سناریوی فعلی و مورد انتظار) ناقص است. هدف این تسک ارائه گزینه‌هایی به کاربر برای انتخاب پروژه مرتبط هنگام ثبت یادآوری و اطمینان از صحت ذخیره‌سازی این ارتباط از طریق API بک‌اند است. این منطق باید با سناریوی مورد انتظار کاربر برای ارتباط یادآوری با پروژه‌ها همخوانی داشته باشد. با توجه به کامیت اخیر `67441ae feat(reminder): add reminder task type with Telegram firing + checklist`، یک سیستم یادآوری در حال حاضر وجود دارد که نیاز به توسعه برای پشتیبانی از ارتباط با پروژه‌ها دارد. فایل `backend/app/api/routes/github_import.py` (خطوط 186-202) شامل `list_imported_projects` است که می‌تواند برای دریافت لیست پروژه‌ها توسط ربات تلگرام استفاده شود. همچنین، `backend/app/api/routes/oversight.py` (که `oversight_telegram_compose` را import می‌کند) نقطه ورود احتمالی برای منطق ربات تلگرام است.
