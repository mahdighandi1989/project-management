# [منطق] عدم وجود validation برای ورودی‌های inspector_session

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

مدل inspector_session ورودی‌هایی مانند 'project_id', 'status', 'title', 'role', 'content', 'action_type' را می‌پذیرد، اما هیچ validation مشخصی برای این فیلدها (مانند محدودیت طول، نوع داده، یا مقادیر مجاز) در مستندات ذکر نشده است.

## 💥 پیامد (impact)
ورودی‌های مخرب یا نادرست می‌توانند باعث خرابی دیتابیس، حملات injection (اگر validation مناسب نباشد)، یا ذخیره داده‌های ناسازگار شوند که منجر به خطاهای runtime می‌شود.

## 🛠 پیشنهاد رفع اولیه
از Pydantic models برای validation ورودی‌ها در endpointها استفاده کنید. محدودیت‌هایی مانند حداکثر طول رشته، enum برای status و action_type، و sanitization برای جلوگیری از injection اضافه کنید.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
