# [منطق] عدم مدیریت race condition در oversight_upload_session

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

سرویس آپلود از فایل‌های JSON برای persistence state استفاده می‌کند. در صورت همزمانی چند درخواست برای یک سشن (مثلاً آپلود همزمان چند chunk)، ممکن است race condition رخ دهد و state فایل خراب شود.

## 💥 پیامد (impact)
از دست رفتن داده‌های آپلود، ناهماهنگی state، و احتمال ایجاد فایل‌های orphan که پاکسازی نمی‌شوند.

## 🛠 پیشنهاد رفع اولیه
از یک lock فایل (file lock) یا یک دیتابیس سبک (مانند SQLite از طریق database.py) برای مدیریت state استفاده کنید. همچنین از transactionهای atomic برای به‌روزرسانی state استفاده شود.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
