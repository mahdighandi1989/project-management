# احتمال وجود Secretهای سخت‌کد شده در کد

## Raw Idea

با توجه به وجود فایل .env.example و استفاده از API keys متعدد (OpenAI, Claude, Gemini, DeepSeek, Render, GitHub)، احتمال دارد که برخی از این کلیدها به صورت سخت‌کد شده در کد یا commit history وجود داشته باشند. این یک آسیب‌پذیری امنیتی جدی است.

## Prompt

## ⚠️ یادداشت مهم برای مدل اجراکننده — قبل از شروع بخوان

این پرامپت بر اساس یک **بررسی اولیهٔ خودکار** از repo ساخته شده — ممکن است
حاوی اشتباه، تشخیص نادرست، یا حذف موارد مهم باشد. به‌عنوان منبع نهایی به
آن استناد نکن.

🔍 **مسئولیت تو (مدل اجراکننده):**
- پیش از هر تغییر، خودت ساختار repo، فایل‌های ذکرشده، و وابستگی‌های آن‌ها را
  مستقل بررسی کن.
- اگر تشخیص دادی موقعیت ذکرشده در پرامپت اشتباه است یا فایل دیگری مناسب‌تر
  است، بر اساس قضاوت خودت عمل کن — این پرامپت نمی‌تواند بهانهٔ کار اشتباه
  باشد ("خودت گفتی" قابل قبول نیست).
- اگر معیارهای پذیرش (AC) مبهم/ناقص بودند، بهترین تفسیر را انتخاب کن و در
  commit message توضیح بده.

📦 **اگر کار طولانی است:**
- **خلاصه‌اش نکن.** همه را به‌طور کامل انجام بده.
- اگر یک کامیت گنجایش ندارد، در **چندین کامیت متوالی** انجام بده — ولی
  هیچ بخشی را skip نکن.
- ترتیب کامیت‌ها را منطقی نگه‌دار (foundation → core → integration → tests).
- در آخر یک checklist از همه‌ی کامیت‌ها در PR description بنویس.

## ✅ معیار پذیرش (Acceptance Criteria)
- [ ] هیچ secret سخت‌کد شده‌ای در کد وجود نداشته باشد
- [ ] تمامی secrets از متغیرهای محیطی خوانده شوند
- [ ] فایل .env در .gitignore باشد
- [ ] pre-commit hook برای جلوگیری از commit secrets فعال باشد
- [ ] هیچ تستی fail نمی‌شود (`npm run test` / `pytest`)
- [ ] linter بدون warning عبور می‌کند
- [ ] type-check موفق است (`tsc --noEmit` / `mypy`)

## 🪜 مراحل اجرایی پیشنهادی
1. اجرای ابزارهای اسکن secret مانند truffleHog یا git-secrets روی کل تاریخچه git. حذف هرگونه secret یافت شده و چرخش (rotate) کلیدهای آسیب‌دیده. افزودن pre-commit hook برای جلوگیری از commit secrets.

## 💡 نمونه‌های قبل/بعد
**قبل: secret در کد**

_قبل:_
```
OPENAI_API_KEY = "sk-..."  # hardcoded in config.py
```

_بعد:_
```
OPENAI_API_KEY = os.getenv("OPENAI_API_KEY")  # read from environment
```

## 📤 خروجی مورد انتظار
تغییر کد در فایل‌های مرتبط، commit یا PR جدید با پیام واضح، و عبور تمام معیارهای پذیرش.

## 🧪 دستورات اعتبارسنجی
- `trufflehog git --since-commit HEAD~10 .`
- `git log --all -p | grep -E '(sk-|ghp_|AIza)'`

## ⚠️ ریسک‌ها و موارد احتیاط
حذف secrets از تاریخچه git نیاز به force push دارد که ممکن است برای همکاران مشکل ایجاد کند. همچنین چرخش کلیدها ممکن است باعث قطع سرویس شود.
