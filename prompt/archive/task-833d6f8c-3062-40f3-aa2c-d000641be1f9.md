# Analysis page uses hardcoded fallback data instead of real API responses

## Raw Idea

The frontend analysis page (`frontend/src/app/analysis/page.tsx`) and model-profiles page (`frontend/src/app/model-profiles/page.tsx`) both contain extensive hardcoded fallback data (defaultProfiles, defaultLeaderboard) that are used when the backend API returns empty or fails. This masks the fact that the backend `/api/models/profiles` and `/api/models/leaderboard` endpoints may not be returning real data. The fallback data includes fake scores (e.g., GPT-4 with 92.5 overall score) that are never actually computed from real analysis runs. This creates a misleading user experience where users see plausible-looking profiles even when no actual analysis has been performed.

## 🪜 مراحل اجرایی پیشنهادی
1. Remove all hardcoded fallback data from both frontend pages. Instead, show a clear empty state with instructions to run an analysis first. _بعد:_
```
if (data.success && data.profiles?.length > 0) { setProfiles(data.profiles); } else { setProfiles([]); showEmptyState(); }
```
