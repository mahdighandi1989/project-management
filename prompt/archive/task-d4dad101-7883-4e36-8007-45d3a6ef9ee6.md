# عدم بررسی انقضای توکن JWT در تابع validate_token

## Raw Idea

تابع validate_token در سرویس احراز هویت، ادعای 'exp' (expiration time) توکن JWT را بررسی نمی‌کند. این به مهاجمی که یک توکن منقضی شده (مثلاً از طریق لاگ سرور یا کلاینت) به دست آورده، اجازه می‌دهد تا به طور نامحدود از آن استفاده کند. این یک نقص امنیتی بحرانی است که تمام endpointهای محافظت‌شده با JWT را تحت تأثیر قرار می‌دهد.

    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except JWTError:
        raise HTTPException(status_code=401, detail='Invalid token')
