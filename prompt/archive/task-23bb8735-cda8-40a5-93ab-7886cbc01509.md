# استفاده از SessionLocal مستقیم در endpointها بدون مدیریت صحیح lifecycle

## Raw Idea

در چندین endpoint (analysis.py خطوط 117, 280, 300, 315, 348; github_import.py خطوط 137, 155; project_health.py خطوط 222-226) از SessionLocal() به صورت مستقیم استفاده شده و در بلوک try/finally بسته می‌شود. این الگو در برابر خطاهای غیرمنتظره (مثل قطع شبکه) آسیب‌پذیر است و ممکن است connection leak ایجاد کند. همچنین در analysis.py خط 117 یک session ساخته می‌شود (analysis_db) که در finally بسته نمی‌شود — فقط در خط 123 به DeepAnalysisService پاس داده شده و هیچ close() روی آن فراخوانی نشده است.

## 💡 نمونه‌های قبل/بعد
**جایگزینی SessionLocal با Depends**

_قبل:_
```
db = SessionLocal()
try:
    ...
finally:
    db.close()
```

_بعد:_
```
async def get_analysis_reports(..., db: Session = Depends(get_db)):
    ...
```
