# عدم تطابق نسخه‌های وابستگی‌های Python بین requirements.txt و فایل‌های lock

## Raw Idea

فایل requirements.txt شامل وابستگی‌های اصلی است اما هیچ فایل Pipfile یا poetry.lock برای پین کردن دقیق نسخه‌ها وجود ندارد. این باعث می‌شود در محیط‌های مختلف (توسعه، CI/CD، production) نسخه‌های متفاوتی نصب شود که می‌تواند منجر به شکستن کد شود. همچنین در docker-compose.yml و Dockerfile از pip install -r requirements.txt استفاده شده که بدون lock file، reproducibility تضمین نمی‌شود.
