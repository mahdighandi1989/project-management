# CORS پیکربندی بیش از حد باز در backend

## Raw Idea

در `backend/app/main.py`، CORS middleware با `allow_origins=["*"]` یا مقدار بسیار باز پیکربندی شده است. این اجازه می‌دهد هر دامنه‌ای به API دسترسی داشته باشد و درخواست‌های cross-origin ارسال کند. این یک آسیب‌پذیری امنیتی جدی است که می‌تواند منجر به حملات CSRF و نشت داده‌ها شود. شواهد در فایل `backend/app/core/config.py` (خطوط مربوط به CORS_ORIGINS) و `backend/app/main.py` (تنظیمات CORS) دیده می‌شود.
