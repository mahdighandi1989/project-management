# Duplicate health analysis logic in project_health.py and analysis.py

## Raw Idea

Both `backend/app/api/routes/project_health.py` and `backend/app/api/routes/analysis.py` implement nearly identical project analysis flows: file collection, AI model invocation, progress streaming via SSE, and report generation. `project_health.py` (1648 lines) is explicitly marked as deprecated with `Deprecation` headers and a `Sunset` date of Dec 2026, yet it still contains the most comprehensive analysis logic including `_merge_with_existing_issues`, `debug_single_file_test`, and `export_health_data`. `analysis.py` (707 lines) has a newer streaming endpoint (`/run-stream`) but lacks the issue merging and debug capabilities. This duplication causes maintenance overhead, inconsistent behavior (e.g., different file extension filters), and risks that bug fixes in one are not ported to the other.
