# [منطق] عدم اشاره به transaction safety در عملیات هم‌زمان

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

در توضیحات هیچ اشاره‌ای به استفاده از transaction یا lock برای عملیات هم‌زمان (مثلاً آپلود chunkها و ذخیره پیام‌ها) نشده است. فایل JSON upload_sessions.json ممکن است در معرض corruption باشد.

## 💥 پیامد (impact)
در صورت درخواست‌های هم‌زمان، ممکن است داده‌ها ناسازگار شوند، فایل JSON خراب شود، یا سشن‌ها duplicate شوند.

## 🛠 پیشنهاد رفع اولیه
از database transaction برای عملیات atomic استفاده کنید. برای فایل JSON از file lock (مثلاً fcntl) یا یک دیتابیس سبک (SQLite) به‌جای JSON استفاده کنید.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
