# [منطق] عدم وجود مکانیزم احراز هویت در database.py

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

فایل database.py مسئول پیکربندی اتصال به دیتابیس است، اما هیچ اشاره‌ای به احراز هویت کاربران در سطح دیتابیس یا application نشده است. این فایل فقط به اتصال فیزیکی به SQLite می‌پردازد و pipeline احراز هویت در آن تعریف نشده است.

## 💥 پیامد (impact)
بدون یک لایه احراز هویت مشخص در pipeline، هر درخواست به API می‌تواند به دیتابیس دسترسی داشته باشد. این باعث می‌شود که کل سیستم در معرض دسترسی غیرمجاز قرار گیرد.

## 🛠 پیشنهاد رفع اولیه
یک dependency برای احراز هویت (مانند JWT verification یا session-based auth) در FastAPI اضافه کنید و آن را به تمام endpointهای مرتبط با inspector_session اعمال کنید. database.py باید فقط مسئول اتصال باشد و احراز هویت در لایه بالاتر مدیریت شود.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
