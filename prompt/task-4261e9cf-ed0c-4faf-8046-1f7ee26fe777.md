# فعال‌سازی «اسکن موردی عمیق» به صورت پشت‌پرده در Inspector chat - بازرس ویژه

## Raw Idea

[TASK] فعال‌سازی «اسکن موردی عمیق» به صورت پشت‌پرده در Inspector chat — بدون UI جدید، با همان فیلدهای موجود session
# 🎯 هدف
قابلیت selective deep scan که در commits قبلی (dba4ae5, fb586b7, 5e2d6b6,
507c345) برای مرکز نظارت ساخته شد، باید **علاوه بر آن** (نه به جایش) به
عنوان موتور پشت‌پردهٔ Inspector chat فعال شود — به‌گونه‌ای که هر پیام
کاربر در chat که قصد بررسی/اصلاح کد دارد، خودکار از همان منطق دقیق
استفاده کند. این یعنی Inspector به جای یک‌شات با ۳KB فایل و بدون import
graph (که الان است)، با همان عمق scan کار می‌کند — ولی بدون اینکه ظاهر
Inspector تغییری بکند.
# ⚠️ ابهام‌زدایی‌های قطعی (به دستور صریح کاربر)
1. **selective scan در مرکز نظارت دست‌نخورده می‌ماند.** هیچ‌چیز از آنجا
   حذف یا منتقل نمی‌شود. این task **افزودنی** است نه جابجایی.
2. **هیچ دکمه/toggle/modal جدید در UI بازرس ویژه اضافه نمی‌شود.** نه
   دکمهٔ «اسکن موردی»، نه toggle «حالت عمیق»، نه section picker.
3. **UI فعلی Inspector کافی است**: textbox پیام، backend logs panel،
   console logs panel، address bar (frontend_url/page_url)، screenshots
   (در حالت visual_debug). همه چیز همین‌جاست.
4. خروجی scan **هرگز به OversightTask DB نمی‌رود**، فقط به InspectorMessage
   در همان session.
5. دکمه‌های موجود اجرای task و «اعمال همه تغییرات» (که در Inspector
   پیاده شده) باید روی این proposals کار کنند.
# 🧠 نگاشت context موجود Inspector → پارامترهای scan
این بنیادی‌ترین قسمت task است. هیچ ورودی جدیدی از کاربر گرفته نمی‌شود —
همه چیز از contextای که Inspector از قبل دارد استخراج می‌شود:

|---|---|

| `custom_paths` | استخراج خودکار از: (الف) URL پیام/page_url نگاشت‌شده به فایل route مربوطه — مثل `/oversight` → `frontend/src/app/oversight/page.tsx`؛ (ب) stack trace هایی که در backend_logs/console_logs اخیر هست؛ (ج) target_files از linked task اگر session linked است؛ (د) فایل‌هایی که نام آن‌ها صریحاً در پیام کاربر آمده |

# 🚦 تشخیص intent — کِی scan اجرا شود، کِی نه
نه هر پیام Inspector باید scan را trigger کند. مثلاً «سلام»، «این کلاس
چی کار می‌کنه؟» یا «خروجی این endpoint چیه؟» نیاز به scan ندارند.
برای جلوگیری از کندی غیرضروری، یک **intent classifier سبک** قبل از scan
اجرا شود:
## گزینهٔ پیاده‌سازی (انتخاب کدنویس بر اساس simplicity):
- **A) Heuristic ساده**: regex/keyword detection روی پیام کاربر.
  کلیدواژه‌های trigger:
  - فارسی: «بررسی کن»، «اصلاح کن»، «درست کن»، «این باگ»، «مشکل»،
    «خطا»، «این کار نمیکنه»، «refactor»، «بهینه»، «این رو ببین»،
    «این چرا»، «اعمال کن»، «پیاده کن»
  - انگلیسی: `fix`, `bug`, `broken`, `not working`, `investigate`,
    `refactor`, `improve`, `apply`, `implement`
  - + هر وقت stack trace در backend_logs اخیر هست
  - + هر وقت visual_debug screenshots اخیراً capture شده‌اند
- **B) AI classifier**: یک LLM call کوچک (با مدل سریع، مثل haiku) با
  prompt «این پیام درخواست اصلاح/تحلیل کد است یا گپ ساده؟ Y/N»
**توصیه**: گزینه A را اول پیاده کن (ارزان، فوری). اگر در عمل false-positive
زیاد داد، با یک خط تغییر به B تبدیل شود (سرویس قابل تعویض باشد).
اگر intent = scan نیست → flow فعلی Inspector chat بدون تغییر اجرا شود.
اگر intent = scan است → flow جدید زیر اجرا شود.

def _extract_focus_keywords(text: str) -> List[str]:
    """استخراج اسم‌های مهم برای semantic search."""
    if not text:
        return []
    tokens = re.findall(r"[\w\-\u0600-\u06ff]{3,}", text.lower())
    stops = _STOPWORDS_FA | _STOPWORDS_EN
    out, seen = [], set()
    for t in tokens:
        if t in stops or t.isdigit():
            continue
        if t not in seen:
            seen.add(t)
            out.append(t)
    return out[:12]

def _extract_focus_keywords(text: str) -> List[str]:
    """استخراج اسم‌های مهم برای semantic search."""
    if not text:
        return []
    tokens = re.findall(r"[\w\-\u0600-\u06ff]{3,}", text.lower())
    stops = _STOPWORDS_FA | _STOPWORDS_EN
    out, seen = [], set()
    for t in tokens:
        if t in stops or t.isdigit():
            continue
        if t not in seen:
            seen.add(t)
            out.append(t)
    return out[:12]

def _extract_keywords_from_notes(text: str) -> List[str]:
    import re
    stops = {"این", "که", "از", "به", "در", "می", "the", "and", "for", "with"}
