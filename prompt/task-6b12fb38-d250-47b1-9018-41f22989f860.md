# [منطق] عدم coherence بین database.py و oversight_upload_session در مدیریت state

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

database.py از SQLite با SQLAlchemy برای ذخیره‌سازی استفاده می‌کند، در حالی که oversight_upload_session از فایل JSON (upload_sessions.json) برای persistent state استفاده می‌کند. این دو مکانیزم state management با هم هماهنگ نیستند. اگر یک سشن آپلود در JSON ذخیره شود اما در دیتابیس نباشد (یا برعکس)، inconsistency ایجاد می‌شود. همچنین migration در database.py فقط به اضافه کردن ستون‌های جدید اشاره دارد و هیچ اشاره‌ای به migration برای جدول upload_sessions ندارد.

## 💥 پیامد (impact)
در صورت crash یا restart سرور، state سشن‌های آپلود ممکن است با state دیتابیس هماهنگ نباشد. این می‌تواند منجر به از دست رفتن داده‌های آپلود شده یا duplicate آپلود شود.

## 🛠 پیشنهاد رفع اولیه
یکپارچه‌سازی state management: یا تمام state را در دیتابیس ذخیره کنید (با اضافه کردن جدول UploadSession به مدل‌ها) و فایل JSON را حذف کنید، یا از یک مکانیزم دوطرفه sync استفاده کنید. همچنین migration برای جدول upload_sessions به database.py اضافه شود.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
