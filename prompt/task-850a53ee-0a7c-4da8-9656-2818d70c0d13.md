# [منطق] عدم coherence بین database.py و inspector_session.py در مدیریت migration

## Raw Idea

## 📋 شرح ناسازگاری
در pipeline `auth` یک ناسازگاری منطقی پیدا شد:

database.py مسئول migration است و از sqlite3 مستقیم برای افزودن ستون‌های جدید استفاده می‌کند، در حالی که inspector_session.py از SQLAlchemy ORM استفاده می‌کند. این دو روش ممکن است با هم ناسازگار باشند و باعث از دست رفتن داده یا خطاهای migration شوند.

## 💥 پیامد (impact)
اگر migration مستقیم sqlite3 با schema تعریف شده در SQLAlchemy هماهنگ نباشد، ممکن است ستون‌ها ناهماهنگ شوند، migrationها شکست بخورند، یا داده‌ها از بین بروند.

## 🛠 پیشنهاد رفع اولیه
از یک ابزار migration یکپارچه مانند Alembic استفاده کنید که با SQLAlchemy سازگار است. متد migrate_db را با Alembic جایگزین کنید تا schema به صورت خودکار و هماهنگ مدیریت شود.

## 🤔 چرا مهم است
coherence issue یعنی دو بخش کد فرض‌های ناسازگار دارند — معمولاً نشانه‌ی refactor ناتمام یا feature flag rot است. این کلاس bug ها در test معمولی پیدا نمی‌شوند چون unit test ها در silo اجرا می‌شوند.
